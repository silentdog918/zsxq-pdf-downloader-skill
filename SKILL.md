# 知识星球 PDF 批量下载助手

## 概述

本技能通过 Chrome 浏览器自动化工具，从知识星球（zsxq.com）的群组帖子中批量下载 PDF 附件。知识星球是一个付费知识社区平台，用户（特别是投资研究领域）经常在上面分享研报PDF。

## 前置条件

1. **用户必须已在 Chrome 中登录知识星球账号**——本技能无法帮用户登录（涉及密码操作属于禁止行为）
2. **需要 Chrome 浏览器自动化工具**（Claude in Chrome MCP）
3. **用户需要告知**：目标群组名称、要下载的日期范围

## 核心工作流

### 第一步：确认目标

向用户确认以下信息（如果对话中尚未明确）：
- 要下载哪个（些）群组的 PDF？
- 下载哪些日期的内容？（今天、最近N天、具体日期等）
- 下载文件保存到哪里？（默认使用用户的 Downloads 文件夹）

### 第二步：导航到目标群组

1. 获取浏览器 tab 上下文（`tabs_context_mcp`），如果没有可用 tab 就创建一个
2. 导航到 `https://wx.zsxq.com`（知识星球网页版）
3. 等待页面加载，截图确认是否已登录
4. 如果未登录，提示用户先手动登录，等待用户确认后再继续
5. 在页面中找到目标群组并点击进入

### 第三步：定位目标日期的帖子

知识星球的帖子是按时间倒序排列的。需要通过滚动来找到目标日期的帖子。

**关键技巧——页面滚动：**
知识星球页面的普通 scroll 事件可能不生效。必须使用 JavaScript 方式滚动：

```javascript
window.scrollBy(0, 800)
```

通过 `javascript_tool` 执行上述代码来向下滚动页面。每次滚动后等待1-2秒让内容加载，然后截图检查当前可见的帖子日期。

**日期定位策略：**
- 截图后查看帖子的日期标签
- 如果当前可见帖子的日期晚于目标日期，继续向下滚动
- 如果已经滚动到目标日期之前的帖子，说明已经过头了，目标区域在上方

### 第四步：逐个下载 PDF

对于每个包含 PDF 附件的帖子，执行以下步骤：

1. **识别 PDF 附件**：在帖子中找到 PDF 文件图标或文件名链接
2. **点击 PDF**：点击文件名或图标，会弹出预览/下载对话框
3. **找到下载按钮**：在弹出的对话框中，用 `find` 工具搜索"下载文件"按钮
4. **点击下载**：点击"下载文件"按钮开始下载
5. **关闭对话框**：下载启动后，关闭预览对话框（按 Escape 键或点击关闭按钮）
6. **等待**：等待1-2秒确保下载启动
7. **移动到下一个 PDF**：重复以上步骤

**下载按钮的位置：** 知识星球的 PDF 预览弹窗中，"下载文件"按钮通常在弹窗的工具栏区域。如果用 `find` 找不到，可以尝试用 `read_page` 查看弹窗的交互元素。

### 第五步：验证下载结果

下载完成后：
1. 挂载用户的 Downloads 文件夹（如果尚未挂载）
2. 列出最近下载的 PDF 文件
3. 向用户报告下载结果：成功下载了多少个文件，列出文件名

## 常见问题处理

### 内容保护 / 无法下载

部分知识星球群组设置了内容保护，限制网页端下载文件（仅允许通过手机 App 下载）。

**识别标志：**
- 点击 PDF 后没有出现"下载文件"按钮
- 弹窗中显示"请在App中查看"或类似提示

**应对策略：**
- 立即通知用户该群组启用了内容保护
- 建议用户通过手机 App 下载这些文件，然后上传给你继续处理
- 不要反复尝试——这是群主的设置，无法绕过

### 网络请求 503 错误

通过 `read_network_requests` 监控下载状态时，可能看到 503 状态码。这**不一定代表下载失败**——浏览器可能会自动重试并最终成功下载。

**正确做法：**
- 不要仅凭网络请求状态码判断下载是否成功
- 通过检查 Downloads 文件夹中是否出现新文件来确认
- 如果文件确实没有出现，再尝试重新下载

### 页面加载缓慢

知识星球加载新内容可能较慢，尤其是在滚动加载更多帖子时。

**应对：**
- 每次滚动后等待 2 秒
- 如果截图显示加载指示器，再等待 2-3 秒
- 如果长时间无新内容加载，尝试再次滚动

### 重复下载

如果同一个 PDF 被多次点击下载，浏览器会自动添加 `(1)`、`(2)` 等后缀。这不影响文件内容，但会占用空间。

**预防：**
- 下载前先记录已下载的文件名
- 使用有序的方式逐个处理帖子，避免重复点击

## 效率建议

- **批量操作时保持节奏**：点击 → 等待弹窗 → 下载 → 关闭弹窗 → 下一个。不要急于连续点击。
- **优先处理当天/最新的帖子**，因为它们在页面顶部，不需要滚动。
- **如果帖子数量很多**（超过20个PDF），建议每下载10个后暂停确认一下下载文件夹，确保一切正常。
- **对于多个群组**，逐个群组处理，每个群组处理完毕后向用户汇报进度。
